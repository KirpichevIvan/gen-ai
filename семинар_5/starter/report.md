## Диагностика ошибок

### 1. Инструмент вернул отсутствие данных

Трасса:

```json
{"run_id":"88958de2-ad28-47b0-9be2-a4cfd499d6af","ts":"2026-06-21T05:40:27","step":1,"call":"get_inflation","args":{"year":2026,"month":5},"obs":{"error":"нет данных ИПЦ на 2026-05"}}
{"run_id":"88958de2-ad28-47b0-9be2-a4cfd499d6af","ts":"2026-06-21T05:40:29","step":2,"call":"get_inflation","args":{"year":2026,"month":4},"obs":{"error":"нет данных ИПЦ на 2026-04"}}
{"run_id":"88958de2-ad28-47b0-9be2-a4cfd499d6af","ts":"2026-06-21T05:40:33","step":3,"call":"get_inflation","args":{"year":2026,"month":3},"obs":{"year":2026,"month":3,"cpi_yoy":6.8,"source":"rosstat_csv"}}
```

Причина: в CSV есть данные только до марта 2026, а вопрос про текущую реальную ставку провоцирует запрос последних месяцев.

Что исправить: добавить в описание `get_inflation` правило "если текущего месяца нет, пробуй последний доступный месяц" или сделать отдельный инструмент поиска последнего доступного наблюдения.

### 2. Исчерпан лимит шагов

Трасса:

```json
{"run_id":"d3f086b6-ee42-413b-bdde-8b179e8d7b7b","ts":"2026-06-21T05:42:08","step":1,"call":"get_inflation","args":{"year":2026,"month":5},"obs":{"error":"нет данных ИПЦ на 2026-05"}}
{"run_id":"d3f086b6-ee42-413b-bdde-8b179e8d7b7b","ts":"2026-06-21T05:42:08","step":1,"final":null,"error":"исчерпан лимит шагов max_iter=1"}
```

Причина: диагностический запуск с `--max-iter 1` не оставил агенту шага на восстановление после отсутствующих данных и расчет через `calculate`.

Что исправить: для multi-hop вопросов держать `max_iter >= 4`, а при ошибке инструмента не считать прогон завершенным без попытки альтернативного периода.

### 3. Неправильная нормализация даты моделью

Трасса:

```json
{"run_id":"641e7c47-4e16-40d9-9947-4520779aacbb","ts":"2026-06-21T05:44:50","step":1,"call":"get_fx_rate","args":{"currency":"USD","on_date":"2022-12-01"},"obs":{"currency":"USD","date":"2022-12-01","rate":60.8803,"source":"cbr_live"}}
{"run_id":"641e7c47-4e16-40d9-9947-4520779aacbb","ts":"2026-06-21T05:44:50","step":1,"call":"get_fx_rate","args":{"currency":"USD","on_date":"2023-01-01"},"obs":{"currency":"USD","date":"2023-01-01","rate":70.3375,"source":"cbr_live"}}
```

Причина: в диагностическом вопросе была некорректная дата `2022-13-01`, но модель молча превратила ее в `2022-12-01`, вместо того чтобы сообщить о невалидном месяце.

Что исправить: добавить в системный промпт запрет на исправление невалидных дат без явного предупреждения, а в trace писать стартовую строку с исходным query для более удобной диагностики таких случаев.
