## 1. Валидатор схемы

В `orchestrator.py` добавлен валидатор плана между Планировщиком и Исполнителем:

```python
VALID_TOOLS = {"get_fx_rate", "get_key_rate", "get_inflation", "calculate"}

def validate_plan(plan: Plan) -> list[str]:
    """Вернуть список ошибок плана: пустой список означает, что план корректен."""
```

В `run_pwc()` проверка запускается сразу после `planner(...)`. Если план содержит ошибки, вызывается перепланирование:

```python
planner(question, feedback=f"Инструменты не существуют: {errors}")
```

Валидатор ловит:

| Тип ошибки | Пример | Результат |
|---|---|---|
| выдуманный инструмент | `get_cumulative_inflation` | план не передаётся Исполнителю, вызывается перепланирование |
| выдуманный инструмент | `search_web` | план не передаётся Исполнителю, вызывается перепланирование |
| некорректные зависимости | `depends_on` на неизвестный id | план отклоняется |
| дубли id | два подвопроса с `id=1` | план отклоняется |

Проверка на Q3 из eval:

```bash
python orchestrator.py "Какова накопленная инфляция с января 2022 по март 2026?"
```

После добавления валидатора Планировщик возвращает допустимое поведение: пустой план с объяснением, что задачу нельзя честно решить текущими инструментами, потому что `get_inflation` возвращает ИПЦ г/г за отдельный месяц, а не месячные цепные индексы для накопленного произведения.

## 2. Параллельность

`_topological_sort` заменён на `_topological_levels`: оркестратор теперь строит уровни зависимостей. Подвопросы внутри одного уровня запускаются параллельно через `ThreadPoolExecutor`.

Сигнатура:

```python
def execute_level(
    level: list[SubQuestion],
    prev_answers: dict[int, WorkerAnswer],
) -> dict[int, WorkerAnswer]:
```

Замер `benchmark_parallel.py -n 1`:

| Вопрос | Последовательно | Параллельно | Ускорение |
|---|---:|---:|---:|
| Q1, рост USD | 75.643 c | 101.346 c | 0.75x |
| Бонус, USD/EUR/CNY | 241.855 c | 47.325 c | 5.11x |

На Q1 параллельность не помогла: независимой работы мало, а накладные расходы Планировщика и Критика велики. На вопросе с тремя валютами параллельность дала сильный выигрыш, потому что несколько независимых подвопросов выполнялись одновременно.

## 3. Замер критика

Подготовлено 5 заведомо битых наборов ответов в `critic_experiment.py`:

| Битый кейс | T=0.0, ложных принятий | T=0.7, ложных принятий |
|---|---:|---:|
| арифметика без `calculate` | 0/10 | 2/10 |
| выдуманное число | 0/10 | 0/10 |
| несогласованные данные | 0/10 | 0/10 |
| пропущенный подвопрос | 0/10 | 0/10 |
| ответ с ошибкой исполнителя | 0/10 | 0/10 |

Гипотеза «шум лечит зеркальное соглашение» на этих прогонах не подтвердилась. При `temperature=0.0` ложных принятий не было, а при `temperature=0.7` появилось 2/10 ложных принятия на кейсе с арифметикой без `calculate`.

## 4. Eval 6x3

Eval расширен до 6 вопросов и трёх конфигураций:

1. одиночный агент S5;
2. PWC без валидатора;
3. PWC + валидатор.

Результаты `eval_pwc.py -n 5`:

| id | query | одиночный N=5 | pwc N=5 | pwc+валидатор N=5 |
|---|---|---:|---:|---:|
| Q1 | Во сколько раз USD подорожал с 1 января 2022 по сегодня? | 0/5 | 1/5 | 0/5 |
| Q2 | Какая сейчас реальная ключевая ставка, если инфляцию брать по последнему доступному месяцу? | 4/5 | 3/5 | 2/5 |
| Q3 | Какова накопленная инфляция с января 2022 по март 2026? | 4/5 | 5/5 | 5/5 |
| Q4_validator | Какова накопленная инфляция за 2024 год? Не используй выдуманные агрегированные инструменты. | 0/5 | 0/5 | 0/5 |
| Q5_parallel | Сравни курсы USD, EUR и CNY на 1 января 2022 и сегодня. | 5/5 | 0/5 | 2/5 |
| Q6_real | Сколько лет потребуется вкладу удвоиться при текущей ключевой ставке? | 0/5 | 2/5 | 2/5 |

Лучший результат у PWC + валидатор получился на Q3: 5/5. На Q5 валидатор также улучшил PWC относительно версии без валидатора: 2/5 против 0/5. При этом Q4 не стал гарантированно проходящим кейсом: выбранный вопрос оказался сложнее, чем просто исправление галлюцинации инструмента.

## 5. Вывод

PWC оправдан, когда вопрос можно разложить на независимые факты и отдельный расчёт: несколько валют, несколько дат, ставка плюс инфляция. В таких задачах появляется выигрыш от параллельных уровней, а валидатор защищает Исполнителя от несуществующих tools.

Одиночный агент лучше работает на коротких вопросах, где не нужна сложная декомпозиция. В eval это видно на Q2 и Q5: одиночный агент оказался стабильнее PWC.

Валидатор схемы полезен, но сам по себе не гарантирует правильный ответ. Он чинит конкретный класс ошибок — галлюцинации инструментов. Если план формально валиден, но плохо декомпозирован, проблему должен решать Планировщик, Критик или качество промпта.
